### 1.1 Purpose of the Document

This document is a reverse-engineered Business Requirements Document (BRD) for the
**ETD Airlines Departure Operations System** (`etd-airlines_1`). It was produced
automatically by analysing the source code, configuration files, UI assets, and unit
tests of the application. Its purpose is to reconstruct what the system was built to
do so that stakeholders can validate, extend, and govern the solution.

This BRD does **not** represent requirements that were agreed upfront; it represents
requirements that are *implied by the existing implementation*. Stakeholders should
review it for accuracy and fill in gaps (especially items labelled **Assumed** or
**Unresolved**) before treating it as a governing document.

a **reference and demonstration application**. Evidence includes: (a) the simulation

## 2. Overall Description

### 2.1 Executive Summary

The ETD Airlines Departure Operations System is a lightweight, browser-based web
application designed to help airport operations staff track and manage estimated
departure times across a set of flights on a departure board. Operators can view
all current flights, update ETDs when delays occur, mark flights as departed or
cancelled, add new flights, and view aggregated statistics including on-time
performance and delay-reason breakdowns. The system is built as a Spring Boot REST
API with a single-page HTML/JavaScript front-end.  
**Confirmed** — derived from `README.md`, controllers, and UI assets.

### 2.2 Business Background / Current-State Overview

Airport departure operations require real-time visibility into flight status and ETD
changes. Operations staff need to track delays across multiple airlines and flights,
communicate changes to other departments, and record actual departure times. Without
a centralised system, operations staff would rely on manual tracking or external
FIDS (Flight Information Display Systems) that cannot be updated by local staff.

This application provides a self-contained, locally hosted departure management tool
that can be run on any machine with Java 17 and a browser.  
**Inferred** — from README deployment instructions and the absence of any external
data integration.

**Reference application caveat:** This system is currently implemented as a
**demonstration and reference application**, not a production deployment. The
simulation job, seed data with fictional routes, and Eclipse IDE project structure
confirm this context. Stakeholders considering production adoption should treat
Section 5 (NFRs) and Appendix C (Issues) as a gap analysis.  
**Inferred** — from system structure and README contents.
